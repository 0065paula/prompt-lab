# Prompt Lab 产品需求设计文档 (PRD)

## 1. 产品定位重构

### 1.1 核心问题
当前 Prompt Lab 功能分散，视觉对比等非核心功能稀释了产品价值。

### 1.2 新定位
**Prompt Lab = 提示词优化工作台**

专注解决一个核心问题：**如何把模糊的提示词转化为高质量的、可复用的提示词**

### 1.3 目标用户
- 频繁使用 AI 但效果不稳定的人
- 想要沉淀优质提示词模板的人
- 需要团队协作管理提示词的人

---

## 2. 核心工作流设计

## 3. 功能模块设计

### 3.1 模块一：提示词诊断室 🩺

**功能描述：**
自动分析当前提示词的质量问题

**核心功能：**

| 诊断维度 | 检查内容 | 输出 |
|---------|---------|------|
| 清晰度 | 指令是否明确无歧义 | 清晰度评分 + 改进建议 |
| 结构化 | 是否有步骤/层次 | 结构优化建议 |
| 完整性 | 上下文是否充足 | 缺失要素提示 |
| 可执行性 | AI能否理解执行 | 潜在问题预警 |

**界面设计：**
- 左侧：原始提示词输入框
- 右侧：诊断报告卡片
  - 综合评分（1-10分）
  - 问题列表（带严重程度标识）
  - 一键优化按钮

---

### 3.2 模块二：智能优化器 ✨

**功能描述：**
基于诊断结果，AI 自动生成优化建议

**核心功能：**

```
优化策略库
├── 结构化策略
│   ├── 添加步骤分解
│   ├── 添加角色设定
│   └── 添加输出格式
├──  clarity 策略
│   ├── 消除歧义
│   ├── 添加示例
│   └── 添加约束条件
└── 高级策略
    ├── 添加验证机制
    ├── 添加错误处理
    └── 添加迭代反馈
```

**交互设计：**
1. 显示多个优化方案（策略 A/B/C）
2. 每个方案显示：
   - 预览效果
   - 预期改进点
   - 适用场景
3. 一键应用 / 手动调整

---

### 3.3 模块三：A/B 测试台 🧪

**功能描述：**
对比不同版本提示词的效果

**核心功能：**

| 功能 | 说明 |
|-----|------|
| 版本对比 | 原始 vs 优化后的并排对比 |
| 测试运行 | 用相同输入测试两个版本 |
| 效果评分 | AI 自动评估输出质量 |
| 人工标注 | 用户标记哪个版本更好 |

**界面设计：**
- 三分屏布局
  - 左：版本 A 提示词 + 输出
  - 中：版本 B 提示词 + 输出
  - 右：对比分析报告

---

### 3.4 模块四：迭代历史 📚

**功能描述：**
记录提示词的所有版本变迁

**核心功能：**

```
版本树结构
v1.0 初始版本
└── v1.1 添加结构化
    └── v1.2 添加示例
        └── v2.0 重大重构
            └── v2.1 微调约束
```

**功能点：**
- 版本对比（diff 视图）
- 回滚到任意版本
- 标注里程碑版本
- 导出版本变更日志

---

### 3.5 模块五：模板工坊 🎨

**功能描述：**
保存和管理优化后的优质提示词

**核心功能：**

| 功能 | 说明 |
|-----|------|
| 模板库 | 按场景分类（写作/编程/分析/创意） |
| 标签系统 | 多维度标签（难度/场景/效果） |
| 快速复制 | 一键复制到剪贴板 |
| 导入导出 | JSON/Markdown 格式 |

## 4. 砍掉的功能

### 4.1 确认移除

| 功能 | 移除原因 |
|-----|---------|
| **视觉对比** | 与提示词优化核心无关 |
| **图片上传/叠加** | 属于设计领域，非文本优化 |
| **截图对比** | 偏离核心用户场景 |

### 4.2 保留但简化

| 原功能 | 新形态 |
|-------|--------|
| PromptLaboratory | 整合为「优化工作流」 |
| IterationHistory | 简化为「版本历史」 |
| PromptTemplates | 整合为「模板工坊」 |

## 5. 技术架构调整

### 5.1 组件重构

```
src/
├── components/
│   ├── DiagnosisRoom.vue      # 提示词诊断室 (新)
│   ├── Optimizer.vue          # 智能优化器 (新)
│   ├── ABTestLab.vue          # A/B 测试台 (替代 VisualComparison)
│   ├── VersionHistory.vue     # 版本历史 (重构 IterationHistory)
│   └── TemplateStudio.vue     # 模板工坊 (重构 PromptTemplates)
├── composables/
│   ├── usePromptDiagnosis.ts  # 诊断逻辑
│   ├── usePromptOptimizer.ts  # 优化逻辑
│   ├── useVersionControl.ts   # 版本管理
│   └── useABTesting.ts        # A/B 测试
└── stores/
    └── prompt.ts              # 提示词状态管理
```

### 5.2 数据模型

```typescript
// 提示词版本
interface PromptVersion {
  id: string
  content: string
  version: string
  createdAt: Date
  diagnosis: DiagnosisResult
  parentId?: string        // 用于版本树
  tags: string[]
}

// 诊断结果
interface DiagnosisResult {
  score: number
  clarity: number
  structure: number
  completeness: number
  issues: Issue[]
  suggestions: Suggestion[]
}

// A/B 测试记录
interface ABTestRecord {
  id: string
  versionA: PromptVersion
  versionB: PromptVersion
  testInput: string
  outputA: string
  outputB: string
  winner?: 'A' | 'B' | 'draw'
  createdAt: Date
}
```
